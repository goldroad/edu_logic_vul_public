# 功能测试说明

## 已实现的功能

### 1. 学生取消订单接口
- **接口地址**: `POST /edu/student/cancel-order`
- **功能**: 允许学生取消状态为PENDING的订单
- **权限验证**: 只能取消属于自己的订单
- **状态检查**: 只能取消待支付状态的订单

### 2. 自动关闭过期订单定时任务
- **执行频率**: 每分钟执行一次
- **关闭条件**: 15分钟前创建且状态仍为PENDING的订单
- **自动操作**: 将过期订单状态更新为CANCELLED
- **日志输出**: 记录关闭的订单信息

## 测试方法

### 测试取消订单功能
1. 登录学生账号 (student/123456)
2. 访问订单管理页面
3. 找到状态为"待支付"的订单
4. 点击"取消订单"按钮
5. 验证订单状态是否更新为"已取消"

### 测试自动关闭过期订单
1. 创建一个新订单但不支付
2. 等待15分钟后观察控制台日志
3. 或者直接修改数据库中订单的create_time为15分钟前
4. 等待1分钟观察定时任务执行情况
